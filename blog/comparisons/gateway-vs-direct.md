# 网关与直连供应商接入的比较

直连供应商的起步成本最低。但当团队开始在意一致性、策略控制、路由能力或跨团队复用时，网关的价值就会迅速显现。

## 直连方式的优势

- 最低的初始抽象成本
- 单个原型项目的最短路径
- 在单服务场景下容易理解

## 网关的优势

- 集中式鉴权与供应商映射
- 一致的请求日志与可观测性
- 无需重写业务代码即可切换上游
- 更安全地落地重试、限额与兜底策略

## 判断维度

| 场景 | 更适合 |
| --- | --- |
| 单个原型项目，且只接一个供应商 | 直连 |
| 多个应用共享 AI 能力 | 网关 |
| 需要路由、策略与故障切换 | 网关 |
| 当前运维成熟度较低，且没有平台团队 | 先直连，尽快复盘 |

## 实际判断

真正的拐点通常不是 API 复杂度，而是团队复杂度。一旦多个应用需要共享同样的 AI 能力，网关就不再是额外架构，而是避免重复策略与运维债务的手段。
