# TokFlux 创作者奖励申请指南

TokFlux 创作者奖励面向正在使用大模型中转站的真实用户。无论你是程序员、设计师，还是经常用 GPT、Claude、Gemini 等模型处理编程、设计、论文和内容创作任务，都可以把自己的使用经验整理成视频或文章参与活动。

优质内容可以获得 TokFlux 平台额度奖励，也可以叠加个人推广链接带来的长期收益。我们更鼓励真实、可复现、能帮其他用户少走弯路的教程、测评、案例和排障记录。

## 支持的创作形式

TokFlux 创作者奖励支持两类内容：视频和文章，重点鼓励能帮助用户完成真实任务的教程、测评、案例和排障经验。

视频适合展示完整流程，例如从注册、创建 API Key、配置模型路由，到在客户端里完成第一次请求。文章更适合沉淀可复用的教程、排障清单和模型接入经验。

## 视频创作激励

你可以在 B 站、抖音、小红书或其他视频平台发布与 TokFlux 相关的视频，题材包括但不限于：

- 如何把 OpenAI 兼容客户端切换到 TokFlux
- ChatBox、Cherry Studio、NextChat、Dify 等工具的接入教程
- Codex、Claude Code、Cursor 等 AI 编程工具接入大模型中转站的实践
- 大模型 API 中转站的使用体验、稳定性测评与成本对比
- 多模型路由、备用供应商、团队统一 Key 管理等进阶用法

发布后提交申请，经审核通过即可获得 TokFlux 平台额度奖励。特别优秀的内容可获得额外推荐，并进入官方教程、案例或社媒素材池。

## 文章创作激励

你可以在 CSDN、知乎、掘金、博客园、B 站专栏或个人博客发布与 TokFlux 相关的文章。推荐选题包括：

- 零基础接入 TokFlux 的完整图文教程
- 使用 OpenAI SDK 调用 TokFlux 的代码示例
- 不同模型供应商的接入体验与成本比较
- 企业或团队为什么需要统一的大模型中转层
- AI 编程工作流里的模型切换、用量控制与备用线路配置
- 401、429、超时、流式中断等常见问题排查
- 将 TokFlux 接入 Dify、FastGPT、Coze、沉浸式翻译、AI编程等工具的实践记录

优质文章可以获得 TokFlux 平台额度奖励。特别适合长期留存的文章，官方可在征得作者同意后转载到 TokFlux 官方渠道，并保留作者署名与推广信息。

## 申请通道与流程

完成内容发布后，请通过 TokFlux 控制台工单、客服或官方指定入口提交申请。为了方便审核，建议一次只提交一份作品，并附上所有发布平台链接。

申请内容建议包含：

- 作品标题
- 作品链接
- 发布平台
- TokFlux 用户 UID 或账号邮箱
- 是否允许官方转载
- 是否需要保留个人推广链接或邀请码

奖励通常会在 1 到 2 个工作日内完成审核。遇到节假日或申请量较大时，审核时间可能顺延。

## 内容基本要求

所有申请奖励的内容都需要满足以下条件：

- 内容必须与 TokFlux、大模型 API 中转、OpenAI 兼容接入、模型路由或相关工具集成有关。
- 作品需要长期公开留存，不能发布后立即删除、隐藏或频繁更换链接。
- 内容应体现真实使用过程，包含可验证的步骤、截图、代码、配置项或问题分析。
- 可以植入自己的推广链接或邀请码，但不能以误导性标题、虚假承诺或夸大收益吸引用户。
- 视频建议在标题、简介或标签中包含 `TokFlux`、`大模型中转站`、`OpenAI 兼容接口` 等关键词。
- 文章建议结构清晰，至少包含使用场景、操作步骤、注意事项和结果验证。

## 奖励评定标准

最终奖励会综合内容质量、阅读或播放数据、平台分发情况以及是否便于官方长期复用来评定。奖励不以积分发放，而是发放为 TokFlux 平台额度；额度按美元等值口径计算，可用于后续模型调用。

### 视频奖励公式

视频奖励以基础奖励额度为核心，再叠加播放、多平台发布和官方复用增益：

```txt
视频总奖励额度 = 基础奖励额度
               + 基础奖励额度 × 播放增益
               + 基础奖励额度 × 多平台发布增益
               + 基础奖励额度 × 官方复用增益
```

示例：某视频基础奖励额度为 `$8`，申请时播放量为 800，对应 `+100%` 播放增益；同时同步发布到小红书，对应 `+10%` 多平台发布增益；若未被官方复用，则官方复用增益为 `0%`。最终奖励为 `$8 + $8 × 100% + $8 × 10% = $16.8` 平台额度。

### 文章奖励公式

文章奖励以基础奖励额度为核心，再叠加阅读、多平台发布和官方复用增益：

```txt
文章总奖励额度 = 基础奖励额度
               + 基础奖励额度 × 阅读增益
               + 基础奖励额度 × 多平台发布增益
               + 基础奖励额度 × 官方复用增益
```

示例：某文章基础奖励额度为 `$5`，申请时阅读量为 180，对应 `+50%` 阅读增益；同时发布到 3 个文章平台，对应 `+20%` 多平台发布增益；若被官方收录到帮助中心，对应 `+20%` 官方复用增益。最终奖励为 `$5 + $5 × 50% + $5 × 20% + $5 × 20% = $9.5` 平台额度。

### 基础奖励额度

基础奖励额度由作品类型和质量评分共同决定：

```txt
基础奖励额度 = 类型基础上限 × 质量评分 ÷ 100
```

| 创作类型 | 基础奖励额度范围 | 类型基础上限 |
| --- | --- | --- |
| 视频 | `$3 - $20` | `$20` |
| 文章 | `$1 - $8` | `$8` |

质量评分满分为 100 分。完整教程、真实演示、清晰排版、有效截图、代码示例和可验证结果都会提高质量评分。

| 评分项 | 分值 | 说明 |
| --- | --- | --- |
| 原创性 | 25 | 是否来自真实使用经验，而不是搬运、洗稿或模板化内容 |
| 实用性 | 25 | 是否能帮助用户完成接入、对比、排障、选型或具体任务 |
| 完整度 | 20 | 是否包含准备、配置、验证结果、注意事项和必要上下文 |
| 表达质量 | 15 | 视频的标题、封面、节奏、字幕，或文章的结构、排版、代码块是否清晰 |
| 真实验证 | 15 | 是否提供截图、录屏、请求结果、错误信息或可复现的操作证据 |

示例：一篇文章质量评分为 75 分，则基础奖励额度为 `$8 × 75 ÷ 100 = $6`。一个视频质量评分为 80 分，则基础奖励额度为 `$20 × 80 ÷ 100 = $16`。

特别优秀、可长期作为官方教程或案例使用的作品，可以突破类型基础上限，由人工复核确定。

| 维度 | 视频重点 | 文章重点 |
| --- | --- | --- |
| 原创性 | 真实录制和讲解，不搬运套模板 | 来自真实接入或项目经验 |
| 实用性 | 能帮助用户完成接入、对比、排障或选型 | 能直接复现步骤或解决具体问题 |
| 完整度 | 包含准备、配置、验证和注意事项 | 包含场景、步骤、代码、截图和结果 |
| 表达质量 | 标题、封面、节奏、配音、字幕清晰 | 标题、结构、表格、代码块易读 |
| 长期价值 | 适合被反复观看和转发 | 适合被搜索收录和长期引用 |

### 视频播放增益

播放增益按申请审核时的公开播放量计算。发布超过 1 个月后再申请的作品，默认不再计算播放增益。

| 播放量范围 | 增益比例 |
| --- | --- |
| 0 - 100 或发布超过 1 个月 | +0% |
| 100 - 500 | +50% |
| 500 - 3000 | +100% |
| 3000 以上 | +200% |

### 文章阅读增益

阅读增益按申请审核时的公开阅读量计算。发布超过 1 个月后再申请的作品，默认不再计算阅读增益。

| 阅读量范围 | 增益比例 |
| --- | --- |
| 0 - 30 或发布超过 1 个月 | +0% |
| 30 - 300 | +50% |
| 300 以上 | +100% |

### 多平台发布增益

| 创作类型 | 增益比例 | 平台要求 |
| --- | --- | --- |
| 视频 | +10% | 除主发布平台外，同时发布到抖音、快手、小红书、视频号等至少 1 个平台 |
| 文章 | +20% | 在 CSDN、知乎、掘金、博客园、B 站专栏、个人博客等平台中至少发布 3 个有效链接 |

### 官方复用增益

| 复用方式 | 增益比例 |
| --- | --- |
| 未复用 | +0% |
| 收录到官方教程、帮助中心或案例页 | +20% |
| 官方账号转载、联合发布或重点推荐 | +50% |

## 加分项

满足以下条件的作品，会在同等质量下获得更高优先级：

- 同一作品在多个平台同步发布，并保持链接长期有效。
- 内容覆盖真实开发工具，例如 OpenAI SDK、Codex、Claude Code、Cursor、Dify、FastGPT、NextChat、ChatBox、Cherry Studio 等。
- 文章或视频包含完整的错误排查路径，而不只是展示成功结果。
- 对成本、稳定性、延迟、模型选择做了清晰且负责任的说明。
- 允许官方转载、二次分发或纳入帮助中心，同时保留作者署名。

## 反作弊要求

为了保证奖励公平，以下情况不会通过审核：

- 直接复制他人作品、洗稿、批量替换品牌词或高度模板化内容。
- 使用明显虚假的播放量、阅读量、评论量或互动数据。
- 发布后立即删除、隐藏、改成无关内容或重复提交同一作品。
- 内容与 TokFlux 或大模型 API 中转无关，只是象征性提及品牌名。
- 诱导用户误解价格、功能、模型能力或服务承诺。
- 大量使用 AI 生成内容但没有真实验证、截图、配置和作者经验补充。

如果作品需要修改，可以在原链接基础上持续完善后再提交。我们更鼓励真实、可复用、能帮用户少踩坑的内容，而不是一次性堆量。

## 奖励复审

如果你对奖励结果有疑问，可以在同一工单或同一申请入口中发起复审。复审会重新查看内容质量、数据表现和平台留存情况。若确认奖励存在偏差，会补发差额；若结果无误，也会给出可改进方向。
